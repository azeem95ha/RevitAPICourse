AI-Driven Revit Automation

This course empowers BIM professionals to automate Revit without spending years learning to code. By leveraging Large Language Models (LLMs), you will learn to generate, debug, and deploy Python scripts for Dynamo and pyRevit.

Course Overview

Learning Objectives

This course is designed to take you from a Revit user to a proficient automator. You will not just learn *what* to do, but *how* to think like a developer, using AI as your co-pilot to build powerful tools.

Prompt Effectively

Learn to ask AI for functional Revit API code using the "Context Triangle" framework.

Debug & Fix

Master the "Iterative Debugging" loop to identify and fix common AI hallucinations and errors.

Replace "Node Spaghetti"

Move beyond complex Dynamo graphs by replacing them with clean, efficient Python nodes.

Deploy Custom Tools

Build and deploy professional add-ins to your company's Revit ribbon using pyRevit.

Required Software

To follow along with the course, you will need access to the following tools. This stack is the modern standard for Revit automation.

  • Revit 2024 or later (for Python 3 support)
  • Dynamo (Pre-installed with Revit)
  • pyRevit (Latest Version)
  • An account for an advanced LLM (ChatGPT Plus, Claude Pro, etc.)
  • VS Code (Recommended for editing script files)

Module 2: AI-Powered Dynamo Scripting

In this module, you'll apply your new AI skills to Dynamo. The goal is to stop relying on finding the right nodes and instead use AI to write powerful, compact Python scripts that do the work for you.

import clr
clr.AddReference('RevitServices')
from RevitServices.Persistence import DocumentManager
clr.AddReference('RevitAPI')
from Autodesk.Revit.DB import Transaction

doc = DocumentManager.Instance.CurrentDBDocument
views = UnwrapElement(IN[0])
prefix = "Draft-"

t = Transaction(doc, "Add View Prefix")
t.Start()
for view in views:
    try:
        view.Name = prefix + view.Name
    except:
        pass
t.Commit()

OUT = views

Text-to-Script Workflow

We will practice our new workflow with several hands-on labs. You will be given a problem and will use an LLM to generate the Python script for a Dynamo node.

Prompt: The Bulk Renamer

"Write a Python script for Dynamo. Input 0 is a list of Views. The script should add the prefix 'Draft-' to the View Name."

Key Lessons:
  • Handling lists of elements (`IN[0]`)
  • Using Transactions to modify the model
  • Setting an element's parameter (Name)

Prompt: Geometry Creation

"Create a list of model lines forming a grid pattern based on X and Y inputs."

Key Lessons:
  • Importing Revit API geometry classes
  • Using `for` loops to create multiple objects
  • Needing a Transaction to create new elements

Prompt: Data Mining

"Select all Walls in the active view, get their 'Volume' parameter, and sum the total volume."

Key Lessons:
  • Using `FilteredElementCollector` to find elements
  • Accessing built-in parameters (`BuiltInParameter.HOST_VOLUME_COMPUTED`)
  • Aggregating data and outputting to `OUT`

Common "Gotchas" & Fixes

AI makes common, predictable mistakes. This section covers the top failures you'll encounter and the exact prompts to fix them. Click each "Gotcha" to reveal the solution.

The Problem: You ask for a 5-foot offset, and the code creates an offset 5 units (e.g., 5mm) wide, or it fails. The AI forgets that Revit's API uses internal units (decimal feet).

The Fix (Prompt): "Use `UnitUtils.ConvertToInternalUnits` to handle the units correctly."

The Problem: The AI uses old, deprecated methods like `Parameter.Set()` which fail in modern Revit versions, as it was trained on older data.

The Fix (Prompt): "This is for Revit 2025. Please use the most current API methods."

The Problem: You try to rename 1000 views, and it takes 5 minutes. The AI put the `Transaction.Start()` and `Transaction.Commit()` *inside* the `for` loop, creating 1000 individual undo states.

The Fix (Prompt): "This is slow. Please 'batch the transaction' by wrapping the entire loop in a single transaction for performance."
